# Security Policy

## Supported Versions

This project is pre-1.0. Security fixes should target the current `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports. Use GitHub private vulnerability reporting if it is enabled on the repository, or contact the repository owner privately.

Include:

- A clear description of the issue
- Steps to reproduce
- Impact and affected versions or commits
- Any suggested mitigation

## Scope

Security-sensitive areas include authentication, session handling, dependency vulnerabilities, database access, and any future external data ingestion jobs.
